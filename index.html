Featured System

Sentinel - AI News Intelligence

Problem

Information overload is a constant challenge. Traditional news aggregators simply pull in headlines, offering volume without context or synthesis. They fail to understand nuance, connect related narratives, or provide a reliable way to interact deeply with complex subjects without hallucinating facts.

Solution

Sentinel is a multi-agent AI system designed to solve this by clustering news articles, verifying citations, and synthesizing cohesive narratives. Beyond basic aggregation, it allows users to actively interrogate the news corpus using a precision-focused RAG pipeline, providing intelligent, contextualized intelligence.

System Architecture

At its core, the system begins with robust data ingestion from external APIs, immediately followed by semantic deduplication to ensure data quality. The processing is managed by a multi-agent workflow consisting of a Researcher, Editor, and Publisher, each handling distinct phases of synthesis. Data is securely stored using a hybrid approach—PostgreSQL for relational metadata and ChromaDB for vector embeddings. The entire intelligence engine is exposed via a FastAPI backend, heavily utilizing Redis for semantic caching to minimize latency. Everything is monitored with comprehensive observability tools to track system health and agent decision-making.

Key Engineering Challenges

  • Deduplication across diverse news sources
  • Preventing hallucinated citations
  • Latency optimization using Redis semantic caching
  • Cost tracking & intelligent rate limiting

  • Production: Every technical decision from choosing a database to caching strategies is evaluated against latency, scalability, and maintainability tradeoffs.
